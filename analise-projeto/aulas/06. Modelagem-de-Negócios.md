# Modelagem de Negócios

## Processos Organizacionais, BPM e Alinhamento TI × Negócio

---

## 🎯 Objetivos da Aula

Ao final da aula, o aluno será capaz de:

- Compreender o que são **processos organizacionais**
- Entender o conceito e a prática do **BPM**
- Identificar a importância do **alinhamento entre TI e Negócio**
- Representar processos usando **notação BPMN**
- Analisar um **estudo de caso empresarial** real

---

## 📌 O que é Modelagem de Negócios?

**Modelagem de Negócios** é a prática de descrever, analisar e representar graficamente como uma organização funciona.

Ela responde perguntas como:

- Como os processos da empresa funcionam?
- Onde estão os gargalos e ineficiências?
- Como a tecnologia pode melhorar esses processos?

> 💡 Modelar um negócio é essencial **antes** de construir qualquer sistema.

---

## 🏢 Por que Modelar Negócios?

Sem modelagem, o desenvolvimento de software fica **sem direção**.

| Sem Modelagem | Com Modelagem |
|---|---|
| Sistema não atende a empresa | Sistema resolve problemas reais |
| Retrabalho constante | Entregas assertivas |
| Comunicação falha | Alinhamento entre equipes |
| Alto custo de manutenção | Sistemas mais sustentáveis |

# Parte 1

## Processos Organizacionais

---

## 🔄 O que é um Processo?

Um **processo** é um conjunto de **atividades relacionadas** que transforma entradas em saídas com um objetivo definido.

### Estrutura básica de um processo:

```
ENTRADA → [ ATIVIDADES ] → SAÍDA
```

**Exemplo:**
```
Pedido do cliente → [ Receber → Processar → Embalar → Entregar ] → Produto entregue
```

---

## 🏗️ Tipos de Processos Organizacionais

Os processos de uma empresa se dividem em três categorias:

### 1. Processos Primários (ou de Negócio)
Entregam valor direto ao cliente.
- Vendas, produção, atendimento ao cliente

### 2. Processos de Suporte
Apoiam os processos primários.
- RH, TI, contabilidade, jurídico

### 3. Processos de Gestão
Controlam e monitoram a organização.
- Planejamento estratégico, auditoria, controle de qualidade

---

## 🧠 Analogia: Restaurante

Pense em um restaurante:

| Tipo | Exemplo |
|---|---|
| **Primário** | Receber pedido → Preparar → Servir |
| **Suporte** | Comprar ingredientes, contratar funcionários |
| **Gestão** | Controlar estoque, avaliar satisfação dos clientes |

> 💡 Todo negócio tem os três tipos de processo, independente do tamanho.

---

## 📊 Características de um Bom Processo

Um processo bem definido deve ser:

- **Documentado** → registrado e compreensível
- **Medido** → com indicadores (KPIs)
- **Controlado** → monitorado continuamente
- **Melhorado** → otimizado ao longo do tempo

> ⚠️ Processos mal definidos geram retrabalho, prejuízo e insatisfação do cliente.

---

## 🔍 Como Identificar Processos em uma Empresa

Para mapear os processos de um negócio:

1. **Entrevistar** os responsáveis por cada área
2. **Observar** como as atividades são realizadas na prática
3. **Analisar documentos**: formulários, sistemas, relatórios
4. **Questionar**: o que entra? o que sai? quem participa?

> 💡 Na Análise de Sistemas, esse levantamento é a base para o desenvolvimento de qualquer software.

# Parte 2

## BPM — Business Process Management

---

## 📋 O que é BPM?

**BPM (Business Process Management)** é uma **disciplina de gestão** que tem como objetivo:

> "Melhorar continuamente os processos de negócio de uma organização."

Ele não é apenas uma ferramenta — é uma **abordagem completa** que envolve:

- Modelagem de processos
- Automação
- Monitoramento
- Melhoria contínua

---

## 🔁 O Ciclo do BPM

O BPM funciona em um ciclo contínuo de 5 fases:

```
  Modelagem → Implementação → Execução
      ↑                              ↓
  Otimização ← Monitoramento ←────────
```

| Fase | Descrição |
|---|---|
| **Modelagem** | Mapear e desenhar os processos |
| **Implementação** | Configurar sistemas e regras |
| **Execução** | Rodar o processo no dia a dia |
| **Monitoramento** | Medir desempenho com indicadores |
| **Otimização** | Melhorar com base nos dados |

---

## 📐 BPMN — A Linguagem do BPM

**BPMN (Business Process Model and Notation)** é a notação padrão para representar processos.

É como uma linguagem gráfica universal que qualquer pessoa — técnica ou não — consegue entender.

### Elementos básicos do BPMN:

| Símbolo | Nome | Uso |
|---|---|---|
| ⭕ | Evento de início | Onde o processo começa |
| ⬛ | Tarefa | Uma atividade realizada |
| 🔷 | Gateway | Decisão ou divisão de fluxo |
| ⭕ (duplo) | Evento de fim | Onde o processo termina |
| → | Fluxo de sequência | Ordem das atividades |

---

## 🖊️ Exemplo de Processo em BPMN

**Processo: Aprovação de pedido de compra**

```
[ Início ]
    ↓
[ Funcionário preenche pedido ]
    ↓
[ Gestor analisa pedido ]
    ↓
  ◇ Aprovado?
   ↙        ↘
[ Sim ]    [ Não ]
  ↓            ↓
[ Gerar  ]  [ Notificar
  ordem  ]    funcionário ]
  ↓            ↓
       [ Fim ]
```

> 💡 Todos os processos de uma empresa podem (e devem) ser representados em BPMN.

---

## 🧰 Pools e Lanes no BPMN

Para representar **quem faz cada atividade**, o BPMN usa:

### Pool
Representa uma **organização ou participante** do processo.

### Lane (Raia)
Divide o pool em **departamentos ou papéis**.

```
┌─────────────────────────────────┐
│         EMPRESA XYZ (Pool)      │
├──────────────┬──────────────────┤
│  Financeiro  │      TI          │
│  (Lane)      │    (Lane)        │
│  [ Aprovar ] │  [ Configurar ]  │
└──────────────┴──────────────────┘
```

---

## 📏 Tipos de Gateways no BPMN

Os gateways controlam o **fluxo do processo**:

| Gateway | Símbolo | Comportamento |
|---|---|---|
| **Exclusivo (XOR)** | ◇ X | Apenas um caminho é seguido |
| **Paralelo (AND)** | ◇ + | Todos os caminhos ocorrem ao mesmo tempo |
| **Inclusivo (OR)** | ◇ O | Um ou mais caminhos podem ser seguidos |

**Exemplo XOR:**
> Se o cliente pagar → gera nota fiscal; senão → envia lembrete

**Exemplo AND:**
> Após aprovação → notifica estoque E financeiro simultaneamente

---

## 🛠️ Ferramentas para BPM/BPMN

| Ferramenta | Característica |
|---|---|
| **Bizagi Modeler** | Gratuita, muito usada no mercado |
| **Camunda** | Open source, permite automação |
| **Lucidchart** | Online, fácil de usar |
| **draw.io** | Gratuita e integrada ao VSCode/Google Drive |
| **Microsoft Visio** | Pago, amplamente corporativo |

> 💡 Para esta aula, recomenda-se o **draw.io** ou **Bizagi** para exercícios práticos.

---

## 📈 Benefícios do BPM

Empresas que adotam BPM obtêm:

- ✅ Maior **eficiência operacional**
- ✅ **Redução de custos** por eliminação de retrabalho
- ✅ **Transparência** nos processos internos
- ✅ Melhor **qualidade** dos produtos e serviços
- ✅ Base sólida para **automação e digitalização**

> 🔑 BPM é o ponto de partida para a **transformação digital** de qualquer empresa.

# Parte 3

## Alinhamento TI × Negócio

---

## 🤝 O Problema Clássico

Por que tantos sistemas de TI **fracassam** mesmo sendo tecnicamente corretos?

> Porque foram construídos **sem compreender o negócio**.

### Cenários comuns de desalinhamento:

- Sistema entregue no prazo, mas **não resolve** o problema do usuário
- TI desenvolve funcionalidades que o **negócio nunca pediu**
- Negócio solicita algo que a **TI não consegue entregar** com os recursos disponíveis
- Mudanças de estratégia deixam sistemas **obsoletos rapidamente**

---

## 🎯 O que é Alinhamento TI × Negócio?

**Alinhamento estratégico** é garantir que os objetivos e investimentos em TI estejam **diretamente conectados** aos objetivos do negócio.

```
Estratégia       Sistemas
de Negócio  ←→  de TI
```

> 💡 TI não deve existir por si mesma — ela deve **servir ao negócio**.

---

## 🗺️ Modelo de Henderson & Venkatraman (SAM)

O **Strategic Alignment Model (SAM)** é o modelo mais reconhecido de alinhamento TI × Negócio.

Ele possui 4 domínios:

| Domínio | Descrição |
|---|---|
| **Estratégia de Negócio** | Onde a empresa quer chegar |
| **Infraestrutura de Negócio** | Como a empresa opera internamente |
| **Estratégia de TI** | Como a TI posiciona seus serviços |
| **Infraestrutura de TI** | Sistemas, redes, dados e processos de TI |

> O alinhamento ocorre quando esses quatro domínios estão **em harmonia**.

---

## 🔗 Como Promover o Alinhamento

Para alinhar TI e Negócio na prática:

1. **Incluir TI no planejamento estratégico** da empresa
2. **Criar comitês** com representantes de negócio e TI
3. **Usar linguagem comum** (evitar jargões técnicos com o negócio)
4. **Modelar processos** antes de desenvolver sistemas (BPM)
5. **Priorizar projetos de TI** com base no valor para o negócio
6. **Medir resultados** com indicadores de negócio, não apenas técnicos

---

## 📊 Indicadores de Alinhamento (KPIs)

Como medir se TI e Negócio estão alinhados?

| Indicador | O que mede |
|---|---|
| **Satisfação dos usuários** | Se o sistema atende às necessidades |
| **% de projetos no prazo** | Capacidade de entrega da TI |
| **Tempo de resposta dos sistemas** | Impacto operacional da TI |
| **ROI dos projetos de TI** | Retorno financeiro dos investimentos |
| **Tempo de resolução de incidentes** | Qualidade do suporte |

---

## 🧩 O Papel do Analista de Sistemas

O Analista de Sistemas é o **elo entre negócio e TI**:

```
  NEGÓCIO             ANALISTA           TI
  (O que quer)  →  (Interpreta)  →  (Como fazer)
                   (Documenta)
                   (Valida)
```

**Competências necessárias:**
- Entender processos de negócio
- Modelar com BPM/BPMN
- Comunicar com clareza
- Traduzir requisitos em soluções técnicas
- Garantir que o sistema entregue valor real

# Parte 4

## Estudo de Caso: Loja Virtual "MercadoFácil"

---

## 🏪 Contexto da Empresa

**MercadoFácil** é uma loja virtual de médio porte que vende produtos eletrônicos.

**Situação atual (problemas):**

- Pedidos são recebidos por **WhatsApp e e-mail**
- Controle de estoque feito em **planilhas Excel**
- Pagamentos conferidos **manualmente**
- Sem integração entre vendas, estoque e financeiro
- Alta taxa de **erros e atrasos** nas entregas

> 📌 A empresa contratou um Analista de Sistemas para modelar e melhorar seus processos.

---

## 🔎 Passo 1: Levantamento dos Processos Atuais

O analista realizou entrevistas e identificou o processo de venda atual:

```
Cliente envia pedido (WhatsApp/e-mail)
        ↓
Atendente anota manualmente
        ↓
Verifica estoque na planilha
        ↓
        ◇ Tem estoque?
       ↙         ↘
    Sim           Não
     ↓              ↓
Confirma      Informa cliente
pedido        e cancela
     ↓
Registra pagamento manualmente
     ↓
Embala e envia produto
     ↓
Atualiza planilha de estoque manualmente
```

---

## 🚨 Problemas Identificados no Processo Atual

| Problema | Impacto |
|---|---|
| Pedidos manuais | Perda de pedidos, erros de digitação |
| Estoque em planilha | Desatualização, conflitos de informação |
| Sem automação de pagamento | Fraudes, confirmações erradas |
| Processo isolado por setor | Retrabalho e falta de visibilidade |
| Sem histórico do cliente | Impossível personalizar atendimento |

> ❌ **Resultado:** Clientes insatisfeitos, perda de vendas e alto custo operacional.

---

## 🗺️ Passo 2: Modelagem do Processo Ideal (AS-IS → TO-BE)

O analista mapeou:

| | Processo Atual (AS-IS) | Processo Ideal (TO-BE) |
|---|---|---|
| Pedido | Manual (WhatsApp) | Plataforma web automatizada |
| Estoque | Planilha Excel | Sistema integrado em tempo real |
| Pagamento | Conferência manual | Gateway de pagamento automático |
| Notificação | Sem notificação | E-mail/SMS automático |
| Relatórios | Planilhas manuais | Dashboard em tempo real |

---

## 📐 Processo TO-BE em BPMN (simplificado)

```
┌──────────────────────────────────────────────────┐
│                 MERCADOFÁCIL                      │
├──────────────┬───────────────┬────────────────────┤
│   Cliente    │   Sistema     │     Estoque/Fin.   │
├──────────────┼───────────────┼────────────────────┤
│ [ Acessa   ] │               │                    │
│ [ loja    ]→ │ [ Recebe    ] │                    │
│              │ [ pedido   ]→ │ [ Verifica       ] │
│              │               │ [ estoque        ] │
│              │               │ [ automatico    ]  │
│              │ ←[ Confirma ] │                    │
│ [ Paga    ]→ │ [ Gateway  ]→ │ [ Registra       ] │
│              │               │ [ pagamento      ] │
│ ← [ Recebe ] │ [ Envia    ]  │                    │
│   [ nota  ]  │ [ notific. ]  │                    │
└──────────────┴───────────────┴────────────────────┘
```

---

## 💻 Passo 3: Alinhamento TI × Negócio

O analista identificou como cada objetivo de negócio se conecta a uma solução de TI:

| Objetivo de Negócio | Solução de TI |
|---|---|
| Aumentar volume de vendas | Plataforma e-commerce escalável |
| Reduzir erros de estoque | Sistema ERP com controle em tempo real |
| Agilizar pagamentos | Integração com gateway (Pix, cartão) |
| Fidelizar clientes | CRM com histórico de compras |
| Tomar decisões baseadas em dados | Dashboard com relatórios automáticos |

> ✅ Cada funcionalidade do sistema existe porque **resolve um problema real de negócio**.

---

## 📊 Passo 4: Indicadores de Sucesso (KPIs)

Após implantar o novo sistema, o MercadoFácil mediu:

| KPI | Antes | Depois |
|---|---|---|
| Tempo médio de processamento do pedido | 45 min | 3 min |
| Taxa de erro no estoque | 18% | 0,5% |
| Satisfação do cliente (NPS) | 42 | 78 |
| Volume de pedidos por dia | 30 | 120 |
| Custo operacional mensal | R$ 12.000 | R$ 7.000 |

> 🚀 A modelagem de negócios + BPM + alinhamento TI levou a uma **transformação real**.

---

## 🧠 Lições do Estudo de Caso

1. **Entender o negócio** é o primeiro passo antes de qualquer solução técnica
2. **Mapear o processo atual (AS-IS)** revela os verdadeiros problemas
3. **Projetar o processo ideal (TO-BE)** orienta o desenvolvimento
4. **Alinhar TI com objetivos de negócio** garante valor real
5. **Medir resultados** com KPIs valida o sucesso da solução

> 💡 Um Analista de Sistemas que só conhece tecnologia está incompleto. É preciso **conhecer o negócio**.

## 💬 Reflexão Final

> *"Não basta automatizar um processo ruim — você apenas fará as coisas erradas mais rápido."*
> — **W. Edwards Deming**

A modelagem de negócios existe para garantir que:

- Entendemos o que precisa ser feito antes de fazer
- Construímos sistemas que **resolvem problemas reais**
- TI e Negócio caminham na **mesma direção**
